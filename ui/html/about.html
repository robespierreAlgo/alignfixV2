AlignFix

A Tool for Augmentation and Refinement of Parallel Corpora

Overview

AlignFix is a sophisticated browser-based application for processing parallel text corpora, computing word alignments, and extracting high-quality phrase pairs for machine translation and linguistic research.

Built entirely with WebAssembly (WASM) and running 100% in your browser, AlignFix provides desktop-class performance without requiring any server infrastructure or data uploads. Your data stays completely private and secure on your local machine.

Key Philosophy: Privacy-first and offline-capable through WebAssembly compilation.

How It Works

  1. 1. Import Your Corpus

    Upload parallel text files (source and target languages) in plain text format, one sentence per line.

  2. 2. Compute Alignments

    Run FastAlign to compute bidirectional word alignments. The algorithm uses expectation-maximization (EM) to learn translation probabilities.

  3. 3. Analyze Quality

    Review alignment quality metrics including confidence scores, agreement rates, and coverage statistics. Filter low-quality alignments and identify problematic sentence pairs.

  4. 4. Extract Phrases

    Extract phrase pairs using heuristic methods based on alignment consistency. Configure phrase length, frequency thresholds, and quality filters.

  5. 5. Fix Errors & Refine

    Identify and fix alignment errors, remove misaligned sentence pairs, correct phrase boundaries, and refine the corpus quality through iterative analysis and manual review.

  6. 6. Export Results

    Export cleaned phrase tables, refined alignments, and quality reports in standard formats (CSV, TMX, Moses).

Performance Capabilities

Benchmark results from complete processing pipeline (tokenization + alignment + phrase extraction + database insertion) on 16-core system:

Corpus Size Tokenization Alignment Phrase Extraction DB Insert Total Time
6K pairs 0.7s 7.2s 5.2s 0.6s ~14 seconds
100K pairs 3.1s 1m 44s 1m 26s 24s ~3.6 minutes
418K pairs 10s 5m 26s 5m 7s 1m 8s ~11.9 minutes
1M pairs 41s 19m 42s 16m 26s 2m 13s ~39 minutes
Note: Benchmarks measured on 16-core system. Performance depends on your hardware (CPU cores, RAM) and browser. Chrome/Edge provide the best WebAssembly performance. Single-threaded (8-core) performance is approximately 2x slower for alignment and phrase extraction.

Privacy & Security

Complete Privacy

  • 100% client-side processing - No data leaves your browser
  • No server uploads - All files processed locally
  • No tracking or analytics - Your work is completely private
  • Offline-capable - Works without internet connection

Data Storage

  • IndexedDB - Browser-native persistent storage
  • Local only - Data stored on your device
  • Full control - Delete projects anytime
  • Export/Import - Backup and restore your work
Sensitive Data Safe: Perfect for confidential documents, proprietary corpora, or any data that cannot be uploaded to external servers.

System Requirements

  • Browser: Chrome 90+, Edge 90+, Firefox 88+, Safari 14+ (Chrome/Edge recommended)
  • RAM: 4GB minimum, 8GB+ recommended for large corpora
  • Storage: 1GB+ free disk space for IndexedDB
  • CPU: Multi-core processor recommended for parallel processing
